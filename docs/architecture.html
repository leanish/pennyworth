a message’s life

Two doors in, one disciplined path through.

Signed consumer envelopes

A question from a person, a normalized webhook event: HMAC over canonical JSON, key resolved per consumer, sender registered in a DynamoDB consumer registry with the message kinds it may send.

Self / scheduler messages

An agent’s own follow-up stages and the recurring tick are unsigned — the input queue is IAM-private to the agent and its scheduler role. A guard rejects unsigned bodies on agents that also accept consumer traffic unless trust is explicitly acknowledged.

the mechanics

Briefly, the clever-but-boring parts.

Idempotency

A conditional claim in DynamoDB: completed, live in-flight, or expired/reclaimable in-flight (an ownership deadline decides liveness). SQS is at-least-once, so duplicates are expected: completed duplicates acknowledge without re-running the skill; still-in-flight duplicates return to the queue for retry; expired claims can be reclaimed by the next delivery.

Stages & delayed self-publish

Multi-step agents (bump-it’s init → breakdown → revisit) self-publish their next stage: publish goes straight to the agent’s own queue; publishDelayed creates a one-shot EventBridge Scheduler schedule whose name derives from {agent, stage, payload} — re-publishing the same logical work dedupes onto the first schedule.

Skills

The unit of agent behavior: a focused instruction set with declared input/output schemas. The runtime stages the agent’s skills (with runtime-bundled fallbacks), mounts the synced working copy, runs the coding agent as a subprocess, and validates output against the schema. Tests swap in a fake runner — no live model needed.

Needs

An agent declares what it touches (github, sqs, s3, …); the runtime wires typed clients and infra derives IAM from the same declaration. Undeclared access fails loudly.

running it locally

The production seams, on a laptop.

run-local CLI

Feeds one message through the real dispatch/skill path with a filesystem catalog and a local working copy; --fake-runner for plumbing smoke tests, the real runner for end-to-end proofs. Known gap: it processes the one message it’s given — no fan-out drain yet.

LocalStack suites

Every core package and agent ships test:integration against real AWS SDK calls routed to LocalStack. The harness fails loudly when LocalStack is unreachable. Honest limitation: LocalStack Community’s Scheduler is CRUD-only (schedules never fire), so the suite verifies the full CreateSchedule contract and simulates the fire byte-for-byte.

Lambda rehearsal

ask-the-code builds its real container image and exercises it through the AWS Runtime Interface Emulator against LocalStack — the closest-to-production path that runs locally.

design decisions

Worth knowing.

No central brain

Coordination happens where humans already coordinate — the ticket and the PR. Agents stay independently adoptable and independently failable.

Deterministic plumbing, judgment at the edge

The runtime never asks a model to do something code can do. That’s the cost story, the predictability story, and the testability story at once.

Descriptor-driven everything

agent.yaml is the single declaration: the runtime validates against it, infra provisions from it, and registry⇄descriptor consistency is enforced by tests.

Fail loud, never silently degrade

Unreachable LocalStack throws; undeclared needs throw; unverifiable envelopes are rejected with a reason taxonomy; the integration gate never silently skips.

Cross-cutting implementation decisions (and their rationale) are recorded in the assumptions record; per-agent decisions live in each agent’s ASSUMPTIONS.md.
