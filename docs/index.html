an attainable — almost obvious — idea

Focus on the product,
not the plumbing.

A small crew of focused AI specialists handles the toil around your code — questions, dependencies, docs, triage — and tees the rest up for your review. People keep the judgment calls; agents take the legwork. Built from boring cloud pieces you already know — which is what makes it so doable.

the problem

Engineering runs on toil.

  • Answering “how does this part work?” for the hundredth time.
  • Chasing dependency updates until something falls behind and bites.
  • Docs that quietly drift away from what the code actually does.
  • Re-triaging the same kinds of problems across systems.

What if a small fleet of AI specialists handled the toil — and you stayed in control?

the idea

One shared runtime. Many focused agents.
Each does one job.

Not autonomous

Every agent only reads, advises, or proposes. A person approves anything that goes live.

One foundation

The shared runtime does the mechanical work in plain, deterministic code — no AI for the plumbing. The coding agent runs only where judgment is needed.

Cloud-native

Hosted as managed AWS services on a shared catalog of your projects — the same access controls, monitoring, and reliability as the rest of your stack.

The gate is the cost trick: irrelevant events never start an agent. Everything consequential lands on a person. The full mechanics — envelopes, idempotency, skills — are on the architecture page.
